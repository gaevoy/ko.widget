Overview

Big amount of rich UI functionality is always become difficult. There is Component Driven Development which aimed to manage code complexity. There are principles how to make components by Uncle Bob:

  • REP - The Release Reuse Equivalency Principle
  • CCP - The Common Closure Principle
  • CRP - The Common Reuse Principle
  • ADP - The Acyclic Dependencies Principle
  • SDP - The Stable Dependencies Principle
  • SAP - The Stable Abstractions Principle

ko.widget is JavaScript library which resolve code complexity during building rich UI in Component Driven Development way. It depends on KnockoutJS and RequireJS.

How To Use

To make ko.widget working your View and ViewModel should be wrapped in Widget class. See Hello Word example:

  • View.htm
  • ViewModel.js
  • HelloWordWidget.js
  • App.js
<div class="example-1">
    <h1>Example 1</h1>
    <div data-bind="text: title"></div>
</div>

define(["knockout"], function (ko) {

    return function ViewModel() {
        this.title = ko.observable("Hello Word");
     
    };

});
define(["ko.widget", "./ViewModel", "text!./View.htm"], function (Widget, ViewModel, View) {

    return function HelloWordWidget() {
        Widget.extend(this, [new ViewModel(), View]);
    };

});
require(["jquery", "App/HelloWordWidget/Widget", "domReady!"], function ($, HelloWordWidget) {

    var app = new HelloWordWidget();
    app.appendTo($("body"));

});

Here is first working widget ready to be used. Note: combine and minimize it using r.js (RequireJS Optimizer tool).
